# Permissions

NetBox uses a fine-grained permissions system that extends Django's built-in model.

## Object Permissions

NetBox permissions are defined per **object type** and per **action**:

- **view** — read any matching object
- **add** — create new objects
- **change** — edit existing objects
- **delete** — permanently remove objects

Permissions can be constrained with **object-level constraints** (a JSON filter that limits which objects the permission applies to).

## Example: Read-only IPAM

Assign a permission with:
- Objects: `ipam.ipaddress`, `ipam.prefix`
- Actions: `view`
- Constraint: _(none)_

Users with this permission can view all IP addresses and prefixes but cannot modify them.

## Tokens & Permissions

API token requests are gated by the same permissions as UI actions. A token generated by a restricted user inherits those restrictions.

## Groups

Assign permissions to groups, then add users to groups. This is the recommended approach for managing permissions at scale.
